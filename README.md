# 🕊️ نظام الوصية الرقمية (Dead Man's Switch)

نظام آلي يراقب "تسجيل دخولك الحياتي" (Check-in). إذا مرّت مدة محددة دون أن تؤكد أنك بخير،
يقوم النظام تلقائياً بإرسال رسالة/فيديو محدد مسبقاً إلى قائمة أشخاص تختارهم عبر البريد الإلكتروني.

**تصميم وتطوير: عمرو** ✍️

---

## ✨ المميزات

- 🖥️ **لوحة تحكم عربية كاملة** (RTL) محمية بكلمة مرور.
- ⏱️ **مؤقّت ذكي**: يعرض المهلة المتبقية بوضوح، وزر "أنا بخير" بضغطة واحدة.
- 📧 **تذكيرات دورية** تصلك بالإيميل مع رابط سري لتأكيد نشاطك.
- 👥 **إدارة المستلمين**: إضافة/إيقاف/حذف من ستصلهم الوصية.
- ⚙️ **تخصيص كامل**: مدة المهلة، دورية التذكير، عنوان الرسالة، نصها، ورابط الفيديو.
- 🔐 **أمان عالٍ**: تشفير كلمة مرور البريد (Fernet)، رابط تحقق عشوائي، مقارنة آمنة للتوكن.
- 📋 **سجل كامل للأحداث**.
- 🧪 **أدوات اختبار**: تشغيل الفحص يدوياً، تنفيذ فوري، إعادة تفعيل، واختبار البريد.
- 🤖 **مجدول مدمج** (APScheduler) — يعمل داخل التطبيق، لا يحتاج cron خارجي.

---

## 🚀 التشغيل المحلي (للتجربة)

```bash
pip install -r requirements.txt
python app.py
```

ثم افتح المتصفح على: `http://localhost:5000`

**كلمة المرور الافتراضية:** `admin` — **غيّرها فوراً** من صفحة الإعدادات.

---

## ⚙️ خطوات الإعداد الأولى

1. سجّل الدخول بكلمة المرور `admin`.
2. اذهب إلى **الإعدادات** وغيّر كلمة مرور اللوحة.
3. املأ **اسمك** و**بريدك** (الذي تصلك عليه التذكيرات).
4. اضبط **مدة المهلة** (افتراضي 60 يوم) و**دورية التذكير** (افتراضي 7 أيام).
5. أدخل **إعدادات البريد (SMTP)**:
   - لـ Gmail: `smtp.gmail.com` / المنفذ `465`.
   - استخدم حساب Gmail **مخصص لهذا الغرض** (ليس حسابك الأصلي).
   - فعّل **التحقق بخطوتين** ثم أنشئ **App Password** واستخدمه (ليس كلمة مرور الحساب العادية).
     - رابط: https://myaccount.google.com/apppasswords
6. اكتب **محتوى الوصية**: العنوان، النص، ورابط الفيديو (يوتيوب/درايف).
7. أضف **المستلمين** من الصفحة الرئيسية.
8. اضغط **اختبار البريد** للتأكد من وصول الرسائل.

---

## 🧪 كيف تختبر آلية التنفيذ بأمان

1. من الإعدادات، اضبط **مدة المهلة** مؤقتاً على يوم واحد.
2. من الصفحة الرئيسية، **لا** تضغط "أنا بخير".
3. اضغط زر **تنفيذ الآن** (في أدوات الاختبار) واكتب كلمة `تنفيذ` للتأكيد.
4. تحقق من وصول الرسالة للمستلمين (جرّب بريدك أنت أولاً كمستلم تجريبي).
5. اضغط **إعادة تفعيل** لإرجاع النظام لحالته الطبيعية، ثم أعد المهلة إلى 60 يوم.

> 💡 المجدول يعمل تلقائياً كل ساعة في الخلفية، ومنطق النظام يمنع تكرار الإرسال بعد التنفيذ.

## 🌐 النشر على سيرفر 24/7

### الخيار (أ): PythonAnywhere (مجاني — موصى به للبداية)

1. أنشئ حساباً مجانياً على [pythonanywhere.com](https://www.pythonanywhere.com).
2. من تبويب **Files**، ارفع ملفات المشروع (أو استخدم `git clone`).
3. من **Consoles** افتح Bash console ونفّذ:
   ```bash
   pip install --user -r requirements.txt
   ```
4. من تبويب **Web** → **Add a new web app** → اختر **Manual configuration** → Python 3.
5. في إعدادات الـ WSGI file، اجعله يشير إلى `wsgi.py` (المتغير اسمه `application`).
6. من تبويب **Web** → **Environment variables**، اضبط:
   - `SECRET_KEY` = نص عشوائي طويل
   - `ENCRYPTION_KEY` = ولّده بالأمر:
     ```bash
     python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
     ```
   - `PUBLIC_BASE_URL` = `https://اسم-المستخدم.pythonanywhere.com`
7. اضغط **Reload**.
8. **مهم**: الحساب المجاني "ينام". استخدم [UptimeRobot](https://uptimerobot.com) (مجاني):
   - أنشئ مراقب من نوع **HTTP(s)** يستهدف: `https://اسم-المستخدم.pythonanywhere.com/health`
   - فترة الفحص: كل 5 دقائق. هذا يبقي التطبيق والمجدول نشطين.

> بديل مضمون أكثر: من تبويب **Tasks** في PythonAnywhere، أنشئ **Scheduled Task** يومية:
> ```bash
> cd ~/webapp && python -c "import app,scheduler; scheduler.daily_job(app.app)"
> ```

### الخيار (ب): Render (مجاني — لا ينام بنفس مشكلة PythonAnywhere)

1. ادفع الكود إلى GitHub.
2. على [render.com](https://render.com) → **New Web Service** → اربط المستودع.
3. **Build Command**: `pip install -r requirements.txt`
4. **Start Command**: `gunicorn wsgi:application` (أضف `gunicorn` لـ requirements.txt).
5. اضبط نفس **متغيرات البيئة** المذكورة أعلاه.
6. استخدم UptimeRobot أيضاً على `/health` لمنع الخمول في الخطة المجانية.

### الخيار (ج): VPS (DigitalOcean وغيره — تحكم كامل)

- شغّل عبر `gunicorn wsgi:application` خلف `nginx`، أو اضبط `systemd service`.
- المجدول المدمج يعمل طالما البروسيس شغّال.

## 🔐 نصائح أمان مهمة جداً

- ❌ **لا تستخدم** كلمة مرور حساباتك الأصلية. استخدم بريداً مخصصاً + App Password.
- ✅ غيّر كلمة مرور اللوحة (`admin`) فوراً.
- ✅ اضبط `SECRET_KEY` و `ENCRYPTION_KEY` كمتغيرات بيئة في الإنتاج.
- ✅ ابدأ الاختبار بمهلة قصيرة جداً، وتأكد من وصول الرسائل قبل الاعتماد عليه.
- ✅ احتفظ بنسخة احتياطية من `data/app.db` و `data/encryption.key` في مكان آمن.
- ✅ اكتب "إرشادات للورثة" فيها مكان السيرفر وكيفية الوصول إليه أو إيقافه.

## 📁 هيكل المشروع

```
webapp/
├── app.py              # التطبيق الرئيسي والمسارات (routes)
├── config.py           # الإعدادات
├── models.py           # نماذج قاعدة البيانات
├── scheduler.py        # محرك المراقبة والتنفيذ
├── mailer.py           # إرسال البريد
├── crypto_utils.py     # التشفير
├── wsgi.py             # نقطة دخول الإنتاج
├── requirements.txt
├── templates/          # واجهات HTML عربية (RTL)
├── static/css/         # التصميم
└── data/               # قاعدة البيانات + مفتاح التشفير (لا يُرفع)
```

## 🔧 التخصيص

كل شيء قابل للتعديل من **لوحة التحكم** مباشرة دون لمس الكود:
مدة المهلة، دورية التذكير، محتوى الرسالة، رابط الفيديو، المستلمون، وحالة تفعيل النظام.

للتخصيص المتقدم في الكود:
- تعديل شكل الرسائل → `mailer.py` (دالة `_build_html`).
- تعديل منطق التنفيذ/التذكير → `scheduler.py`.
- تعديل التصميم → `static/css/style.css`.
